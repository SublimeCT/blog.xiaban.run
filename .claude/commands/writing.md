## 参数
- `content`: 需要生成的内容摘要
- `filePathWithPosition`: 当前操作的文件路径(可以包含行号), 可选, 默认为当前打开的  `markdown` 文件或当前上下文中的 `markdown` 文件
- `length`: 生成的内容长度, 可选

### 参数验证
- 当 `content` 为空时禁止执行任何操作, 并提示用户输入内容
- 当 `filePathWithPosition` 为空时禁止执行任何操作, 并提示用户提供文件路径

## 任务
1. 直接执行 `git add $filePath` 将当前文件的内容添加到暂存区
2. 读取当前文件的 **全部内容**
3. 根据 **当前文件的内容** / **[写作风格](#写作风格)** / **[写作要求](#写作要求)** 生成文章内容, 并且如果 `length` 不为空, 则根据 `length` 限制生成的内容长度, 否则按实际情况生成长度合理的内容
4. 如果 `filePathWithPosition` 中包含行号, 则在行号后面加入生成的内容, 否则只输出生成的内容
5. 输出统计信息

## 写作风格

### 总体特征

作者（Ryan）的写作风格以**实用性**为核心，融合了技术专业性与个人生活色彩，形成了独特的"程序员-旅行者"双重视角。

### 技术教程类文章

#### 结构特点
- **开篇引入**: 常从实际问题或个人使用场景引入，如"在使用 X 时遇到了 Y 问题"
- **目录导航**: 系列文章会提供目录列表，方便读者跳转
- **渐进式讲解**: 从基础概念 → 安装配置 → 实际应用 → 高级技巧
- **代码示例丰富**: 每个步骤都配有完整的命令或代码块

#### 语言特点
- **技术术语保持英文**: 如 `Slash Commands` / `fish shell` / `Vibe Coding`
- **中文描述通俗**: 使用"纵享丝滑"、"爆火"等网络流行语
- **emoji 运用恰当**: 用 `🔥` / `✨` / `⚠️` 等标记重点和注意事项
- **提示框分级**: 使用 `:::tip` / `:::warning` 区分提示和警告

#### 链接处理
- **外部链接丰富**: 大量引用官方文档、GitHub 仓库等
- **内链串联文章**: 经常链接到自己的其他相关文章
- **推荐服务链接**: 会推荐自己认为性价比高的服务（如 GLM Coding）

#### code 块
- 必须对英文单词增加 `` 语法, 必须对所有文件路径使用这个语法

### 旅行游记类文章

#### 叙事结构
- **前言哲思**: 常从人生感悟切入，如"人无法判断一个瞬间的价值，直到它成为回忆"
- **行程规划**: 详细列出景点、路线、住宿等实用信息
- **图文并茂**: 大量原创照片 + 详细说明
- **历史背景**: 会介绍景点的历史文化背景

#### 文学色彩
- **古诗词引用**: 如"回首向来萧瑟处，归去，也无风雨也无晴"
- **比喻修辞**: "如果人生是一场开放世界游戏，那么旅行就是一个个支线任务"
- **情感真挚**: 对历史事件（如南京大屠杀）表达庄重态度

#### 实用信息
- **高德地图链接**: 每个景点都附上地图链接
- **价格提示**: 如门票价格、交通费用
- **避坑指南**: 分享踩过的坑，如预约方式、商业化程度等

### 语言风格共性

#### 口语化表达
- **第一人称**: 大量使用"我"的视角

- **情绪表达**: "😭" / "😡" / "🤔" 等表情符号辅助情绪传达

#### 中文技术写作特色
- **中英混杂**: 技术名词保持英文，描述用中文
- **标点习惯**: 中英文之间不加空格，保持中文排版习惯

- **重点标记**: 使用 `**粗体**` 标记关键术语
